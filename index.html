1. Getting Started with Taskiba

1.1 What is Taskiba?

Taskiba is a project and work management platform that helps teams plan work, track progress, manage time and costs, collaborate with stakeholders, and keep delivery predictable. It’s designed to work for small teams and scale for multi-department organizations.

1.2 Key Features Overview

  • Multi-Workspace Organization: Separate environments for clients, departments, or ventures.
  • Project Management: Milestones, deadlines, budgets, teams, and reporting.
  • Task Management: Kanban workflows, priorities, assignments, and task views.
  • Time Tracking: Timers, timesheets, approvals, and productivity reporting.
  • Bug Tracking: Capture, classify, assign, and resolve issues with traceability.
  • Financial Tools: Expenses, invoices, and profitability insight.
  • Collaboration: Comments, files, mentions, notifications, and activity logs.

1.3 Who Should Use Taskiba?

  • Project managers, product teams, development teams, agencies
  • Consulting firms, freelancers, SMBs, remote/distributed teams

2. Registration and Login

2.1 User Registration

  1. Open your Taskiba sign up page.
  2. Enter name, email address, password, and confirm password.
  3. Accept terms/privacy and create account.

2.2 Email Verification

  1. Check your inbox (and spam/junk) for the verification email.
  2. Click the secure verification link to activate the account.
  3. If expired, request a new verification email from the login screen.

2.3 Login

  • Use verified email + password to sign in.
  • If MFA/2FA is enabled (admin setting), complete the verification step.

2.4 Forgot Password

  1. Click “Forgot Password”.
  2. Enter your email to receive a reset link.
  3. Set a new strong password and log in again.

3. Workspaces Management

3.1 Understanding Workspaces

Workspaces are independent environments with their own projects, teams, settings, permissions, and financial preferences. They’re ideal for separating clients, departments, or business lines.

3.2 Creating a New Workspace

  1. Go to Workspaces from the sidebar.
  2. Click Create Workspace.
  3. Set workspace name and description, then create.

3.3 Managing Workspace Members

3.3.1 Invite Members

  • Go to Workspace Settings → Members.
  • Invite users by email, choose role, send invitation.
  • Track pending invites, resend, or cancel.

3.3.2 Roles and Permissions

  • Owner: full control, billing, settings, members.
  • Manager: manage projects, tasks, teams, reports.
  • Member: do assigned work, track time, submit expenses.
  • Client: restricted view of progress/deliverables.

3.4 Workspace Settings

  • Workspace information (name, description)
  • Default preferences (currency, date/time formats where applicable)
  • Permission controls and visibility policies

4. Project Management

4.1 Creating Projects

  1. Go to Projects → Create Project.
  2. Give the project a clear name, short description, and choose visibility (workspace-wide or restricted to invited members/clients).
  3. Set start/end dates, estimated hours, and choose a budget model (fixed fee or time & materials).
  4. Pick default billable rate and currency, then create the project.

4.2 Managing Project Team

  • Add users from the workspace and assign roles: Owner (full control), Manager (plan/monitor), Contributor (execute tasks), Client (read-only milestone/deliverable view).
  • Control rate visibility for clients and limit who can log time or approve expenses.
  • Removing a member keeps their historical work but blocks new assignments.

4.3 Planning & Structure

  • Create milestones with target dates and attach tasks to keep delivery scoped.
  • Use task stages to reflect the project workflow (e.g., Brief → Design → Build → QA → Deploy).
  • Set task dependencies so upstream work must finish before downstream tasks can start.

4.4 Budget & Financial Controls

  • Define a planned budget in money or hours; choose whether to include expenses in burn calculations.
  • Turn on alerts when 50/75/90% of budget is reached.
  • Compare Planned vs. Actual using billable time, cost rates, and approved expenses.

4.5 Tracking & Closeout

  • Use the activity feed to review changes, comments, file uploads, and status updates.
  • Monitor velocity and budget burndown from the project dashboard.
  • When delivered, mark the project Closed (locks new time/expenses) or Archived (read-only for audit).

5. Task Management

5.1 Task Basics

Tasks represent the smallest unit of planned work. Each task stores ownership, priority, dates, checklist items, attachments, and the time/expenses logged against it.

5.2 Creating Tasks

  1. Open Tasks (or a project) → Create Task.
  2. Add title, description, project, and optional milestone/dependency.
  3. Set priority, start/due dates, estimate, and assign one or more owners.
  4. Choose billable vs. non-billable, then create. The task appears in the first workflow column.

5.3 Managing Tasks

  • Progress: drag cards between stages or update percent done.
  • Checklists & sub-tasks: break the work down and track completion.
  • Files & comments: upload assets, @mention teammates, and pin key decisions.
  • Time & expenses: start a timer or add manual entries directly from the task.

5.4 Task Views

  • Kanban: drag-and-drop through workflow stages.
  • List: fast editing with multi-select, filters, and inline fields.
  • Calendar: schedule by due date and balance workload.
  • My Work: personal queue of assignments across projects.

5.5 Stages & Automation

  • Customize stage names and order per workspace to match your methodology.
  • Optional automation: auto-assign reviewers when a task enters Review; auto-complete when all subtasks are done.

5.6 Productivity Tips

  • Create templates for repetitive task types to reuse descriptions, checklists, and default assignees.
  • Use saved filters (e.g., “Overdue high priority”) to drive standups.
  • Bulk update priority, assignee, or stage from the list view when triaging.

6. Bug Tracking

6.1 Logging Bugs

  1. Create a bug with clear title, environment (device/OS/browser), severity, and reproducible steps.
  2. Attach screenshots or video and capture expected vs. actual results.
  3. Link the bug to the affected project, module, or milestone.

6.2 Triage & Prioritization

  • Use severity/priority to decide order: Blocker → Critical → Major → Minor → Trivial.
  • Tag duplicates and close with reference to the canonical bug.
  • Assign an owner and target fix version or sprint.

6.3 Resolution Workflow

  • Statuses typically move Open → In Progress → In Review → Resolved → Closed.
  • Developers link commits or pull requests; QA verifies reproduction and regression scope.
  • Use comments to log root cause, fix notes, and testing evidence.

6.4 Quality Controls

  • Require attachments (screenshots/logs) for high-severity bugs.
  • Enable mandatory reviewer for resolution of blockers/critical items.
  • Export bug list for release notes or retrospectives.

7. Time Tracking & Timesheets

7.1 Timers

  • Start/stop timers from tasks or the global timer; Taskiba tracks running timers to prevent overlaps.
  • Mark time as billable/non-billable and choose the rate (project default or custom).
  • Pause instead of stopping to keep context when switching meetings or interruptions.

7.2 Manual Time Entries

  • Add entries with date, project/task, duration, notes, and billable flag.
  • Duplicate last week’s entries or create recurring templates for standing meetings.
  • Edit or delete entries before they are approved/locked.

7.3 Timesheets

  • Switch between daily, weekly, monthly, or calendar views to fill gaps quickly.
  • Submit timesheets for approval if your workspace enforces review: Pending → Approved/Rejected.
  • Rejected entries return with reviewer comments; update and resubmit.

7.4 Approvals & Locks

  • Managers can lock a period (e.g., previous week/month) after approval to preserve billing accuracy.
  • Locked entries cannot be edited and feed directly into invoicing and reports.

7.5 Rates, Rounding & Overtime

  • Configure per-user or per-project billable rates; Taskiba uses the rate active at entry time.
  • Optional rounding rules (e.g., 6-minute increments) apply at invoice export.
  • Flag overtime and differentiate it in reports if your policy requires it.

8. Expense Management

8.1 Capturing Expenses

  1. Click Expenses → Add Expense.
  2. Enter category, vendor, amount, tax, currency, and date.
  3. Attach a receipt (image/PDF) and add a short note for reviewers.
  4. Link to a project and task to keep profitability accurate.

8.2 Categories & Taxes

  • Use consistent categories (travel, software, subcontractor, etc.) for clean reporting.
  • Apply tax rates when needed; Taskiba stores tax separately for invoicing.

8.3 Approval Workflow

  • Submit expenses for approval; managers can approve, reject with comments, or request edits.
  • Approved expenses become available for invoicing; rejected items remain visible to the submitter.

8.4 Reimbursements & Billing

  • Mark an expense as reimbursable to an employee or billable to a client.
  • Billable expenses can be pulled into invoices alongside billable time.

9. Invoicing

9.1 Invoice Sources

  • Include approved billable time (respecting rates/rounding) and approved billable expenses.
  • Optionally add fixed-fee line items or discounts.

9.2 Creating an Invoice

  1. Go to Invoices → Create.
  2. Select client/project, issue date, due date, and currency.
  3. Add billable time/expenses via “Add from Timesheets/Expenses”, then adjust quantities or rates if needed.
  4. Insert terms & notes (e.g., net 15, wire details) and save as Draft or Send.

9.3 Sending & Payments

  • Email the invoice PDF with a secure payment link when online payments are configured.
  • Accept payments through your enabled gateways; payments update the invoice balance automatically.
  • Record offline payments (bank transfer, cash) to keep AR accurate.

9.4 Status Lifecycle

Draft → Sent → Partially Paid → Paid → Overdue → Cancelled

  • Overdue reminders can be sent automatically based on due date.
  • Void or cancel to remove an invoice from collections while keeping history.

10. Reporting & Analytics

10.1 Standard Reports

  • Time & Billing: billable vs. non-billable hours, utilization, and invoice readiness.
  • Project Health: milestone status, budget burn, schedule variance.
  • Team Workload: allocation by person, upcoming over/under-load.
  • Expenses: spend by category, reimbursable vs. billable.
  • Bug Quality: open/closed trend, SLA compliance, severity mix.

10.2 Filters, Grouping & Export

  • Filter by workspace, project, assignee, date range, tag, or client.
  • Group results (e.g., by project then assignee) to surface patterns.
  • Export to CSV/PDF for sharing or further analysis.

10.3 Scheduling & Sharing

  • Save report presets with filters applied.
  • Schedule email delivery (e.g., Monday 9 AM) to stakeholders.
  • Share view-only links with clients if your workspace allows external sharing.

10.4 KPI Definitions

  • Utilization: billable hours ÷ available hours for the selected period.
  • Budget burn: actual cost ÷ planned budget.
  • Cycle time: average time from task start to done.

11. Contact & Support

11.1 Contact Form

  • Open the Contact page, choose a topic (support, billing, feedback), and submit your message.
  • Attach screenshots or files to speed up troubleshooting.

11.2 Response Handling

  • Admins receive new requests in the Contact Requests queue and can assign owners.
  • Status options: Open, Waiting on Customer, Resolved, Archived.
  • Replies are emailed to the requester and stored in the thread for history.

11.3 Escalation

  • Flag urgent issues to notify workspace owners.
  • Link a request to a project task/bug when it becomes a tracked work item.

12. Currency Management

12.1 Default Currency

  • Choose a base currency for the workspace; all budgets, rates, and invoices default to it.
  • Currency symbol and thousand/decimal separators follow your locale settings.

12.2 Multi-Currency Usage

  • Projects can override currency if you bill a client in their local currency.
  • Expenses recorded in another currency should include the exchange rate used.

12.3 Changing Currency Safely

  • Changing the base currency impacts historical reports; do this only at period boundaries and communicate to finance users.
  • Export current invoices before switching to avoid rounding surprises.

13. Notifications

13.1 Notification Types

  • Assignments, mentions, status changes, due/overdue reminders, approvals (time/expenses), and billing events.
  • Choose delivery channels: in-app, email, or both.

13.2 Personal Preferences

  • Each user can mute non-critical events or enable daily digests for quieter inboxes.
  • Set “Do Not Disturb” hours to pause emails overnight.

13.3 Admin Controls

  • Admins can enforce critical notifications (e.g., security, billing) that cannot be muted.
  • Customize email templates and branding in System Settings → Email.

14. Roles & Permissions

14.1 Role Overview

  • Owner: full control over billing, system settings, and all workspaces.
  • Admin/Manager: manage projects, people, approvals, and reports inside assigned workspaces.
  • Member: work on assigned tasks, log time/expenses, view relevant projects.
  • Client: limited read-only access to agreed projects, invoices, and deliverables.

14.2 Permissions Granularity

  • Control who can create/delete projects, approve timesheets/expenses, view rates, or edit invoices.
  • Restrict sensitive settings (SMTP keys, payment gateways, storage) to owners only.

14.3 Best Practices

  • Apply least-privilege: grant only the permissions needed for the role.
  • Use client roles instead of sharing staff accounts; audit trails remain intact.

15. Dashboard

15.1 Overview Widgets

  • Key metrics: active projects, overdue tasks, open bugs, unsubmitted timesheets, and pending approvals.
  • Financial snapshots: billable vs. non-billable hours, budget burn, upcoming invoices.

15.2 Customizing the Dashboard

  • Reorder or hide widgets to focus on what matters to your role.
  • Filter by workspace, project, or assignee to narrow the view.
  • Save a dashboard layout per role (manager vs. individual contributor).

15.3 Action Shortcuts

  • Quick-create task, start timer, or log expense from the dashboard.
  • Click any metric to drill down into the underlying report or list.

16. Settings & Configuration

16.1 Company Settings

  • Keep legal name, address, tax/VAT ID, and contact info updated for invoices and emails.
  • Set default date/time format, first day of week, and timezone for consistent scheduling.
  • Upload your logo for use in PDFs, invoices, and email templates.

16.2 Payment Configuration

  • Enable gateways supported by your deployment (e.g., Stripe, PayPal, bank transfer placeholders).
  • Store API keys securely; test in sandbox mode before switching to live keys.
  • Define payment terms (Net 7/15/30) and auto-reminders for overdue invoices.

16.3 Localization

  • Select default language (where supported) and number/date formats.
  • Customize email templates to reflect local tone and compliance statements.

16.4 Data & Backup

  • Configure backup frequency and retention according to your compliance needs.
  • Request data export (projects, tasks, time, expenses) before making structural changes.

17. System Settings

17.1 Brand Settings

  • Upload light/dark logos and choose accent colors for the app and exported PDFs.
  • Preview email templates after branding changes to ensure readability.

17.2 Email Settings

  • Configure SMTP/provider credentials, sender name, and reply-to address.
  • Send a test email to confirm SPF/DKIM deliverability.
  • Set rate limits to avoid provider throttling on bulk notifications.

17.3 Currency Settings

  • Global currency defaults and formatting rules for billing.

17.4 Payment Settings

  • Turn payment methods on/off, set invoice payment behavior.

17.5 Storage Settings

  • Select local vs. S3-compatible storage; provide bucket name, region, and keys if using S3.
  • Define upload size limits and allowed file types for security.

17.6 ReCaptcha Settings

  • Enable captcha to reduce bot signups and abuse.
  • Store keys securely and test forms.

17.7 ChatGPT Integration

  • Enable AI assistance features (where included in your build).
  • Configure API key and usage limits for cost control.

17.8 SEO Settings

  • Configure metadata (title, description, social sharing image) for public pages.

17.9 Cache Settings

  • Manage caching for performance and clear cache after deploying updates or changing settings.
  • Schedule regular cache purges if you frequently update public pages.
