Taskiba User Manual

A complete guide to setting up Taskiba, managing workspaces, projects, tasks, time tracking, billing, reports, and system settings.

Start reading

1. Getting Started with Taskiba

1.1 What is Taskiba?

Taskiba is a project and work management platform that helps teams plan work, track progress, manage time and costs, collaborate with stakeholders, and keep delivery predictable. It‚Äôs designed to work for small teams and scale for multi-department organizations.

1.2 Key Features Overview

  • Multi-Workspace Organization: Separate environments for clients, departments, or ventures.
  • Project Management: Milestones, deadlines, budgets, teams, and reporting.
  • Task Management: Kanban workflows, priorities, assignments, and task views.
  • Time Tracking: Timers, timesheets, approvals, and productivity reporting.
  • Bug Tracking: Capture, classify, assign, and resolve issues with traceability.
  • Financial Tools: Expenses, invoices, and profitability insight.
  • Collaboration: Comments, files, mentions, notifications, and activity logs.

1.3 Who Should Use Taskiba?

  • Project managers, product teams, development teams, agencies
  • Consulting firms, freelancers, SMBs, remote/distributed teams

2. Registration and Login

2.1 User Registration

  1. Open your Taskiba sign up page.
  2. Enter name, email address, password, and confirm password.
  3. Accept terms/privacy and create account.

2.2 Email Verification

  1. Check your inbox (and spam/junk) for the verification email.
  2. Click the secure verification link to activate the account.
  3. If expired, request a new verification email from the login screen.

2.3 Login

  • Use verified email + password to sign in.
  • If MFA/2FA is enabled (admin setting), complete the verification step.

2.4 Forgot Password

  1. Click ‚ÄúForgot Password‚Äù.
  2. Enter your email to receive a reset link.
  3. Set a new strong password and log in again.

3. Workspaces Management

3.1 Understanding Workspaces

Workspaces are independent environments with their own projects, teams, settings, permissions, and financial preferences. They‚Äôre ideal for separating clients, departments, or business lines.

3.2 Creating a New Workspace

  1. Go to Workspaces from the sidebar.
  2. Click Create Workspace.
  3. Set workspace name and description, then create.

3.3 Managing Workspace Members

3.3.1 Invite Members

  • Go to Workspace Settings ‚Üí Members.
  • Invite users by email, choose role, send invitation.
  • Track pending invites, resend, or cancel.

3.3.2 Roles and Permissions

  • Owner: full control, billing, settings, members.
  • Manager: manage projects, tasks, teams, reports.
  • Member: do assigned work, track time, submit expenses.
  • Client: restricted view of progress/deliverables.

3.4 Workspace Settings

  • Workspace information (name, description)
  • Default preferences (currency, date/time formats where applicable)
  • Permission controls and visibility policies

4. Project Management

4.1 Creating Projects

  1. Open Projects ‚Üí Create Project.
  2. Enter project name, optional description, and visibility.
  3. Set dates (start/deadline), estimated hours, and budget if needed.
  4. Create the project.

4.2 Managing Project Team

  • Add members to the project team and assign roles (owner/manager/member/client visibility).
  • Remove users if needed; historical activity remains preserved.

4.3 Project Features

  • Milestones, tasks, time tracking and progress updates
  • Files, comments, and activity logs
  • Project dashboards and reports

4.4 Project Budget

  • Set planned budget (optional) to compare planned vs actual cost.
  • Track expenses, billable time, and invoice totals.

4.5 Project Activities & Tracking

  • Activity feed shows updates, changes, comments, and progress.
  • Use reports to monitor delivery, time usage, and cost.

5. Task Management

5.1 Understanding Tasks

Tasks are actionable work items linked to projects. Tasks can be organized by stages (Kanban), assigned to users, given priorities, deadlines, and tracked for progress.

5.2 Creating Tasks

  1. Go to Tasks ‚Üí Create Task (or create within a project).
  2. Fill in title, description, project association, optional milestone.
  3. Set priority, start/end dates, and assign members.
  4. Create the task (it appears in the first workflow stage).

5.3 Managing Tasks

  • View: details, progress, comments, files, history.
  • Edit: update description, dates, priority, assignments.
  • Move stages: drag-and-drop between workflow columns.
  • Comments & mentions: collaborate and notify teammates.

5.4 Task Views

  • Kanban: workflow board by stages
  • List: sortable/filterable task list
  • Calendar: view by due date
  • My Tasks: focus on your assignments

5.5 Task Stages

  • Default stages typically include: To Do, In Progress, Review, Done.
  • Stages can be customized per workspace to match team workflows.

6. Bug Tracking

  • Create bug reports with title, description, severity, and reproduction steps.
  • Assign bugs to team members and track status through resolution.
  • Link bugs to projects/milestones and keep history for auditing.

7. Time Tracking & Timesheets

7.1 Timers

  • Start/stop timers on tasks to capture accurate work time.
  • Mark time as billable/non-billable.

7.2 Manual Time Entries

  • Add entries with date, project/task, duration, and notes.
  • Use copy/recurring entries for repeated activities.

7.3 Timesheets

  • Views: daily, weekly, monthly, calendar
  • Submit for approval (optional workflow): pending ‚Üí approved/rejected

8. Expense Management

  • Create expenses with category, amount, date, and attachments.
  • Link expenses to projects for profitability and budgeting.
  • Approval workflows can be enabled depending on role policies.

9. Invoicing

  • Create invoices from billable time and expenses.
  • Send to clients and track status (draft/sent/paid/overdue).
  • Support multiple payment methods depending on configuration.

10. Reporting & Analytics

  • Project progress and milestone completion reports
  • Time and utilization (by member, project, date range)
  • Financial insights: budgets, expenses, invoice totals

11. Contact & Support

  • Use the contact form (if enabled) to submit inquiries.
  • Admins can manage submissions, reply, archive, and track status.

12. Currency Management

  • Set workspace/base currency and symbol formatting.
  • Ensure invoices, budgets, and expenses display consistently.

13. Notifications

  • Configure alerts for assignments, comments, due dates, approvals, and billing events.
  • Use in-app + email notifications depending on settings.

14. Roles & Permissions

  • Define what each role can view/manage (workspace, projects, tasks, billing, system settings).
  • Enforce least-privilege access for security.

15. Dashboard

  • Overview of projects, tasks, due items, activity, and workload.
  • Analytics widgets (progress, time usage, financial summaries).

16. Settings & Configuration

16.1 Company Settings

  • Company profile, address, legal/billing info (as required).
  • Operational defaults (date/time, region where applicable).

16.2 Payment Configuration

  • Enable one or more payment gateways (based on your build).
  • Configure API keys securely and validate in sandbox/test mode first.

17. System Settings

17.1 Brand Settings

  • Upload logo, set brand colors, and control UI identity.
  • Ensure email templates and PDFs match your brand.

17.2 Email Settings

  • Configure SMTP/provider settings.
  • Test sending to ensure deliverability.

17.3 Currency Settings

  • Global currency defaults and formatting rules for billing.

17.4 Payment Settings

  • Turn payment methods on/off, set invoice payment behavior.

17.5 Storage Settings

  • Select local vs S3-compatible storage depending on deployment needs.
  • Set upload limits and allowed file types.

17.6 ReCaptcha Settings

  • Enable captcha to reduce bot signups and abuse.
  • Store keys securely and test forms.

17.7 ChatGPT Integration

  • Enable AI assistance features (where included in your build).
  • Configure API key and usage limits for cost control.

17.8 SEO Settings

  • Configure metadata for public pages (if applicable).

17.9 Cache Settings

  • Manage caching for performance and clear cache when deploying updates.
